3. Authentication
3.1 Obtain an API key
Expo Express issues API keys per merchant account (contact support).
Example: 92829a36-2996-4757-a531-29985bf19037
3.2 Passing the key
Embed the key at the top level for every POST request:
{
  "APIKEY": "YOUR_API_KEY",
  /* endpoint‑specific fields */
}

5. Error Reference
HTTPstatusTypical messageLikely Cause / Fix400400Invalid API keyWrong / inactive APIKEY400400Missing field: CityInvalid payload401401UnauthorizedKey not authorized404404Waybill not foundWrong or stale waybill429429Rate limit exceededThrottle requests500500Internal server errorTransient; retry6. Quick‑start cURL
# List cities
curl -X GET https://developer.expoexpressnp.com/v1/Cities

# Quote 1 kg Document
curl -X POST https://developer.expoexpressnp.com/v1/DomesticRateCalculation \
     -H "Content-Type: application/json" \
     -d @rate-request.json

# Create a shipment
curl -X POST https://developer.expoexpressnp.com/v1/CreateShipments \
     -H "Content-Type: application/json" \
     -d @create-shipments.json
